# 🚀 바이브코딩 교실 — 사이트 설명서

> 중학생을 위한 **AI 바이브코딩 입문** 학습 사이트.
> 제공받은 `the-infinite-grid` 인터랙티브 컴포넌트를 UI로 삼고,
> 1·2·3차시가 아니라 **주제별**로 내용을 재구성했습니다.

---

## 1. 이 사이트는 무엇인가요?

"바이브코딩"은 코드를 한 글자씩 직접 치는 대신, **AI(클로드 코드)에게 우리말로 시켜서** 프로그램을 만드는 새로운 방식입니다. 이 사이트는 그 개념과 실제 흐름(도구 설치 → 사이트 제작 → 데이터·보안 → 배포 → 하네스 엔지니어링)을 **중학생 눈높이**로 설명하고, **직접 만져보는 활동**으로 익히게 만든 학습 사이트입니다.

- **대상**: 중학생 (코딩 경험 없어도 OK)
- **목표**: 개념 이해 + 직접 해보는 경험 + "집에서 진짜로 해보기"까지 연결
- **특징**: 보기만 하는 슬라이드가 아니라, 터미널·퀴즈·게임을 **직접 조작**하며 배움

## 4. 직접 해보는 활동 (인터랙션)

| 활동 | 설명 | 어디서 |
|------|------|--------|
| 🖥️ **가상 터미널** | 진짜 컴퓨터 걱정 없이 `pwd`·`mkdir`·`cd`·`ls`·`claude` 를 쳐보면 실제처럼 반응. 없는 폴더로 이동하면 친절한 오류 안내까지. | 주제 2 |
| ✍️ **프롬프트 빌더** | 빈칸을 채우면 AI에게 보낼 부탁 문장이 실시간 완성 → 복사 버튼. | 주제 3 |
| 🧠 **퀴즈** | 정답을 누르면 즉시 ✓ 표시 + 설명. 다 풀면 점수 요약. | 주제 6, 9 |
| 🧩 **짝 맞추기 게임** | 개념과 비유를 눌러서 짝짓기. 다 맞히면 색종이 효과. | 주제 8 |
| 🔄 **6축 카드** | 하네스의 6가지 축을 눌러서 펼쳐보기. | 주제 9 |
| ✅ **체크리스트** | "집에서 진짜로 해보기" 항목. 체크 상태가 브라우저에 저장됨(localStorage). | 주제 9 |
| ✨ **그리드 히어로** | 마우스를 움직이면 배경 그리드가 밝게 드러나고 무한히 스크롤. | 첫 화면 |

## 5. 수업에서 이렇게 쓰세요 (교사용 팁)

1. **첫 화면**을 띄워 "오늘 우리가 할 건 이거예요"로 분위기 잡기 (커서 움직이는 그리드 시연).
2. **그룹 A**(시작하기)부터 순서대로 진행. 주제 2의 **가상 터미널**은 학생들이 직접 한 줄씩 쳐보게 하면 명령어 공포가 사라집니다.
3. 개념 설명 후 곧바로 **활동**으로 넘어가 "직접 해보기"로 굳히기.
4. 주제 6 **보안 퀴즈**, 주제 8 **짝 맞추기**는 모둠 대항전으로 진행해도 좋아요.
5. 마지막 주제 9의 **체크리스트**를 숙제로 연결 — 집에서 환경 설치 + 첫 작품 만들기.
6. 주제 칩으로 자유 이동이 가능하니, 복습할 때 원하는 주제만 골라 다시 볼 수 있습니다.

## 6. 기술 스택 & 폴더 구조

- **Vite + React 18 + TypeScript**
- **Tailwind CSS** — shadcn 호환 CSS 변수 테마(다크 모드 기본)
- **framer-motion** — 그리드 히어로 애니메이션
- **lucide-react** — 아이콘

```
src/
├── components/
│   ├── ui/
│   │   ├── the-infinite-grid.tsx   ← 제공받은 컴포넌트 (원본 유지)
│   │   └── demo.tsx
│   ├── GridBackground.tsx          ← 그리드 효과 재사용 래퍼(히어로 배경)
│   ├── Terminal.tsx                ← 가상 터미널
│   ├── Quiz.tsx                    ← 퀴즈
│   ├── MatchGame.tsx               ← 짝 맞추기
│   ├── PromptBuilder.tsx           ← 프롬프트 빌더
│   ├── Checklist.tsx               ← 체크리스트
│   ├── AxisCards.tsx               ← 하네스 6축
│   └── Primitives.tsx              ← 카드·콜아웃·스텝 등 공용 UI
├── data/topics.tsx                 ← ⭐ 주제별 콘텐츠 (내용 수정은 여기서!)
├── lib/
│   ├── utils.ts                    ← shadcn cn() 유틸
│   └── confetti.ts                 ← 색종이 효과
├── App.tsx                         ← 히어로 + 주제 네비 + 콘텐츠 조립
└── index.css                       ← Tailwind + shadcn 테마 변수
```

## 8. 왜 React/shadcn 프로젝트로 새로 만들었나요?

제공해주신 `the-infinite-grid` 컴포넌트는 **React + TypeScript + Tailwind(shadcn CSS 변수) + framer-motion**을 전제로 작성돼 있습니다. (`@/lib/utils`의 `cn`, `bg-background`/`text-muted-foreground` 같은 shadcn 토큰, `motion.*` 사용)

기존의 순수 HTML/CSS/JS 사이트로는 이 컴포넌트를 그대로 쓸 수 없어, shadcn 표준 구조를 갖춘 React 프로젝트를 새로 세팅하고 컴포넌트를 `components/ui`에 정석대로 배치했습니다. 그래서 앞으로 `npx shadcn add ...`로 다른 shadcn 컴포넌트도 바로 추가할 수 있습니다.
